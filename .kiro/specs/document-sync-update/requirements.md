# 需求文档 - 文档同步功能更新

## 介绍

根据业务需求讨论,项目质控助手中的文档不是由用户手动上传的,而是从数据中台自动同步过来的。因此需要将前端原型中所有"上传文档"相关的功能和文案改为"文档同步"功能。

## 术语表

- **数据中台**: 企业统一的数据管理平台,负责文档的集中存储和分发
- **文档同步**: 从数据中台自动获取和更新文档的过程,无需用户手动上传
- **项目阶段**: 项目生命周期中的不同阶段,包括立项、招标、实施、验收

## 需求

### 需求 1: 更新招标阶段文档功能

**用户故事:** 作为项目经理,我希望看到文档是从数据中台同步而来的,而不是手动上传的,这样我能清楚了解文档的来源和管理方式。

#### 验收标准

1. WHEN 用户查看招标阶段页面 THEN 系统应显示"文档同步"而不是"上传文档"
2. WHEN 用户点击文档同步按钮 THEN 系统应触发从数据中台同步文档的操作
3. WHEN 文档已同步 THEN 系统应显示"已从数据中台同步"的状态标识
4. WHEN 文档待同步 THEN 系统应显示"点击同步"或"等待同步"的提示文案

### 需求 2: 更新立项阶段文档功能

**用户故事:** 作为项目经理,我希望立项阶段的文档管理方式与招标阶段保持一致,都使用文档同步功能。

#### 验收标准

1. WHEN 用户查看立项阶段页面 THEN 系统应显示"文档同步"而不是"上传文档"
2. WHEN 用户点击文档同步按钮 THEN 系统应触发从数据中台同步文档的操作
3. WHEN 文档已同步 THEN 系统应显示"已从数据中台同步"的状态标识
4. WHEN 文档待同步 THEN 系统应显示"点击同步"或"等待同步"的提示文案

### 需求 3: 更新项目详情页文档功能

**用户故事:** 作为项目经理,我希望项目详情页的文档管理方式也使用文档同步功能,保持与其他页面的一致性。

#### 验收标准

1. WHEN 用户查看项目详情页 THEN 系统应显示"文档同步"而不是"上传"
2. WHEN 文档状态为待处理 THEN 系统应显示"待同步"而不是"待上传"
3. WHEN 文档已完成 THEN 系统应显示"已从数据中台同步"而不是"上传于"
4. WHEN 用户点击同步按钮 THEN 系统应触发从数据中台同步文档的操作

### 需求 4: 保持验收阶段现有实现

**用户故事:** 作为开发人员,我希望验收阶段页面保持现有的"已从数据中台同步"实现,因为它已经符合新的业务逻辑。

#### 验收标准

1. WHEN 检查验收阶段页面 THEN 系统应保持现有的"已从数据中台同步"文案
2. WHEN 验收阶段有新文档 THEN 系统应继续使用数据中台同步的方式

### 需求 5: 更新帮助文档

**用户故事:** 作为用户,我希望帮助中心的文档能准确反映系统的实际功能,避免误导。

#### 验收标准

1. WHEN 用户查看帮助中心 THEN 系统应将"文档上传与解析"相关说明更新为"文档同步与解析"
2. WHEN 用户阅读帮助文档 THEN 系统应清楚说明文档来自数据中台而非手动上传

### 需求 6: 统一视觉和交互设计

**用户故事:** 作为用户,我希望文档同步功能在不同页面有一致的视觉呈现和交互方式。

#### 验收标准

1. WHEN 用户在不同阶段页面查看文档功能 THEN 系统应使用统一的图标和按钮样式
2. WHEN 文档同步按钮被点击 THEN 系统应显示统一的加载状态和反馈信息
3. WHEN 文档同步完成 THEN 系统应显示统一的成功提示
4. WHEN 文档同步失败 THEN 系统应显示统一的错误提示和重试选项
