# 项目质控助手前端设计稿

## 项目说明

这是项目质控助手系统的前端静态设计稿，使用 HTML + Tailwind CSS 实现，包含所有核心页面的视觉设计和交互原型。

## 页面列表

### 核心业务页面（5个）
1. **project-qc-home.html** - 项目质控助手首页（项目概览、统计数据、快速入口）
2. **project-qc-document-upload.html** - 文档上传页面（上传立项、招标、合同文档）
3. **project-qc-extraction-review.html** - 内容校验页面（查看提取内容、人工校验）
4. **project-qc-progress-tracking.html** - 进度跟踪页面（项目时间轴、交付物、付款、人员跟踪）
5. **project-qc-alert-dashboard.html** - 预警看板页面（风险预警、异常提示）

### 辅助页面（2个）
6. **user-settings.html** - 个人设置页面（用户信息、偏好设置）
7. **help-center.html** - 帮助中心页面（使用指南、常见问题）

### 组件和文档（3个）
8. **components/navbar.html** - 统一导航栏组件
9. **notifications.html** - 通知中心页面
10. **design-system.html** - 设计系统文档（参考）

## 如何查看

1. 直接在浏览器中打开任何 HTML 文件即可查看
2. 建议从 `project-qc-home.html` 开始浏览
3. 所有页面都使用 Tailwind CSS CDN，无需安装依赖
4. 查看 `design-system.html` 了解完整的设计规范

## 核心业务流程

```
文档上传 → 内容提取校验 → 数据入库 → 进度跟踪 → 预警展示
    ↓          ↓              ↓          ↑          ↑
  首页  ←  自动校验  ←  人工复核  ←  异常检测  ←  风险分析
```

## 交互功能

### 已实现的交互

#### 通用交互（所有页面）
- ✅ **统一导航栏**：5个主菜单（首页、文档上传、内容校验、进度跟踪、预警看板）
- ✅ **通知中心**：展开/收起，显示3条未读通知
- ✅ **用户菜单**：展开/收起，包含个人设置、帮助中心、退出登录
- ✅ **当前页面高亮**：自动高亮当前页面的导航菜单项
- ✅ **点击外部关闭**：点击页面其他区域自动关闭下拉菜单

#### 首页（project-qc-home.html）
- ✅ **快速入口**：4个功能卡片（文档上传、内容校验、进度跟踪、预警看板）
- ✅ **统计数据**：项目总数、待处理任务、预警数量、完成率
- ✅ **最近项目**：项目列表展示，点击"查看全部"跳转到进度跟踪页面

#### 文档上传（project-qc-document-upload.html）
- ✅ **文档上传**：支持拖拽上传或点击选择文件
- ✅ **文档类型选择**：立项文档、招标文档、合同文档
- ✅ **开始解析**：点击链接跳转到内容校验页面

#### 内容校验（project-qc-extraction-review.html）
- ✅ **项目选择器**：切换不同项目，自动保存到localStorage
- ✅ **Tab切换**：5个Tab（付款计划、时间节点、交付物、人员要求、分包政策）
- ✅ **自动校验结果**：展示AI提取的结构化内容
- ✅ **查看原文档**：每个Tab顶部有"查看原文档"按钮
- ✅ **确认并入库**：点击链接跳转到进度跟踪页面

#### 进度跟踪（project-qc-progress-tracking.html）
- ✅ **项目选择器**：切换不同项目，支持URL参数和localStorage
- ✅ **项目时间轴**：展示项目各阶段的时间节点
- ✅ **交付物跟踪**：展示交付物清单和完成状态
- ✅ **付款跟踪**：展示付款计划和实际付款情况
- ✅ **人员进场跟踪**：展示人员进场情况和异常告警
  - 人员数量统计（约定人数、已进场、缺口、不符数、符合率）
  - 人员进场详情表格（按岗位展示）
  - 人员数量不足告警（黄色）
  - 约定人员不符告警（红色）

#### 预警看板（project-qc-alert-dashboard.html）
- ✅ **预警统计**：总预警数、高风险、中风险、低风险
- ✅ **预警筛选**：支持4个维度筛选（风险等级、预警类型、项目名称、处理状态）
- ✅ **实时筛选**：无需刷新页面，实时显示筛选结果
- ✅ **查看详情**：点击跳转到进度跟踪页面，带上项目ID参数
- ✅ **人员进场异常预警**：
  - 中风险：人员数量不足
  - 高风险：约定人员不符

### 数据持久化

#### localStorage使用
| 键名 | 值类型 | 用途 | 使用页面 |
|-----|-------|------|---------|
| `currentProjectId` | String | 保存当前选择的项目ID | 内容校验、进度跟踪 |

#### URL参数使用
| 参数名 | 值类型 | 用途 | 使用场景 |
|-------|-------|------|---------|
| `projectId` | String | 传递项目ID | 预警看板 → 进度跟踪 |

#### 优先级规则
```
URL参数 > localStorage > 默认值
```

## 设计系统

### 颜色系统
- **主色调**: Blue 600 (#2563eb)
- **成功/完成**: Green 600 (#16a34a) / Green 500 (#22c55e)
- **警告/注意**: Yellow 600 (#ca8a04) / Yellow 500 (#eab308)
- **错误/延期**: Red 600 (#dc2626) / Red 500 (#ef4444)
- **进行中**: Blue 600 (#2563eb)
- **待处理**: Gray 600 (#4b5563)

### 字体系统
- 页面主标题: text-2xl (24px) / font-bold
- 区块标题: text-lg (18px) / font-semibold
- 卡片标题: text-base (16px) / font-semibold
- 正文: text-sm (14px) / font-normal
- 次要文本: text-xs (12px) / font-normal

### 间距系统
- 组件内部间距: p-4 或 p-6
- 卡片间距: gap-4 或 gap-6
- 区块间距: mb-6 或 mb-8
- 页面边距: px-4 sm:px-6 lg:px-8

详细设计规范请查看 `design-system.html`

## 核心功能说明

### 1. 文档上传与解析
- 支持上传立项文档、招标文档、合同文档
- 自动提取关键信息（付款计划、时间节点、交付物、人员要求、分包政策）
- AI自动校验提取内容的准确性

### 2. 内容校验与复核
- 展示AI提取的结构化内容
- 支持人工查看和确认
- 5个Tab分类展示不同类型的内容
- 每个Tab可查看原文档进行对照

### 3. 进度跟踪
- **项目时间轴**：展示项目各阶段的时间节点和完成状态
- **交付物跟踪**：展示交付物清单、计划时间、实际时间、完成状态
- **付款跟踪**：展示付款计划、金额、实际付款情况
- **人员进场跟踪**：
  - 人员数量比对（约定人数 vs 实际进场人数）
  - 人员信息比对（约定人员名单 vs 实际进场人员名单）
  - 人员异常告警（数量不足、约定人员不符）

### 4. 预警看板
- **风险等级分类**：高风险、中风险、低风险
- **预警类型**：材料不一致、逾期风险、交付物缺失、审计风险、人员进场异常
- **多维度筛选**：支持按风险等级、预警类型、项目名称、处理状态筛选
- **人员进场异常预警**：
  - 中风险：人员数量不足
  - 高风险：约定人员不符

## 设计理念

### 聚焦展示，简化操作
项目质控助手是一个**查看和监控工具**，主要功能是**展示风险和预警信息**，实际处理操作在其他系统完成。

### 清晰的信息流转
```
文档上传 → 内容提取校验 → 数据入库 → 进度跟踪 → 预警展示
```

### 减少开发工作量
- 移除复杂的状态管理
- 移除不必要的交互逻辑
- 专注于核心展示功能
- 尽量精简需求，能完成需求即可
